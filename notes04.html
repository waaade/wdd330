Chapter 8: Forms

  • DOM method of accessing a form: document.forms[0]. Or you can search by the name attribute: document.forms.thename.
  • A form Javascript object has a method called elements, it has the contents.
  • Interesting: <input type='image' src='button.png'> This is how you use an image as a button.
  • input.value gets the text inside the field (input is just a name here, could be anything)
  • Javascript validation is useful, it can keep too many server requests from happening, plus it's another layer of protection, though there should always be server side validation as well
  • A good style for errors: <style> .error{ background: #f99; border: #900 1px solid; display: none; } </style>

Chapter 12: Object-Oriented Programming

  • Javascript is a prototype-based language rather than a class-based language. This means that OOP in Javascript involves making an object and then reusing it instead of making a blueprint (class) as in C++ or Java. (Javascript does have classes after ES6 but it uses the prototype model in the background)
  • Parentheses are not required when instantiating a new object using a constructor function. const redDice = new Dice;. If arguments need to provided, then parentheses are required.
  • Different ways to create an object... const anObject = {}; anObject = new Object();
  • Javascript classes are declared similarly to other languages. They are easier to read and use.
  • Static methods: Class.function() instead of instance.function()
  • As you'd expect, the values of individual instances of a class can be overwritten with dot notation
  • Private variables are a matter of scope. Declare one inside a constructor (this is different from the normal this.thing = ) and it can't be accessed without getters and setters
  • Inheritance: use extends (similar syntax to Java)
  • Mixin is a thing that makes a deep copy of an object. It can add properties to an object
  • Smaller objects (fewer properties) are better design. Make big object from smaller ones

Modular Javascript

  • A module is self-contained. It's a piece of code that should serve one purpose.
  • Javascript module files use the keyword export.
  • A module can have one default export which can be used in a file without having to use curly braces
