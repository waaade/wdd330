CSS3 Transforms and Transitions

  • Translate: moves an element
  • To move something along the x axis (45 px to the right in this case): transform: translateX(45px)
  • Scale: It scales the element, of course. transform:scale(1.5, 0.25). Providing only one value scales both x and y.
  • If you want multiple transform functions, put them in one space-separated list
  • Rotate: again, does what you expect. Degrees (deg) is probably the easiest parameter to give it but there are others: grads, radians, turns
  • Skew also uses degrees
  • .class h1 span { transition-property: transform; } Here a transition property is added to the transforms for that particular element.
  • For an element to have a transition it needs a default state and a final state (such as a :hover state). Then you need to add the transition functions.
  • Duration: very self-explanatory, just give it the number of seconds. Timing-function has more particular timing stuff. There's ease, linear, ease-in... You can define your own function too but that's advanced.
  • Delay: a small delay is often a good idea for transitions.
  • Transition shorthand: transition: transform 0.2s ease-out 50ms;
  • Animations let you use keyframes. A keyframe looks like a CSS declaration block. Animations also have similar functions to transitions, but there's more stuff too.

Canvas, SVG, and Drag and Drop

  • Canvas is an API that lets you draw anything with JavaScript. Useful because you don't need to download images.
  • Put Canvas in with html tags, give it an id and class, give it height and width. Probably should give it CSS border too.
  • var context = canvas.getContext("2d"); This returns an object that will let you draw in 2D.
  • strokeStyle, fillStyle (set them to colors). fillRect, strokeRectangle (give them coordinates of where to begin drawing and then height and width)
  • Those are the basic ideas of Canvas but there's a lot more you can do
